## End-to-End Flows

Three implicit user flows, all complete within the placeholder strategy:

### Flow A: Landing → Checkout
1. User visits `harmblocker.bg` → loads `index.html` over HTTPS (Phase 1 config + user action)
2. User reads hero + benefits + pricing (Phase 3 brand-voice copy)
3. User clicks hero or pricing CTA → Stripe Checkout (Phase 4 placeholder → user-swapped URL)
4. Before click, user sees 14-day withdrawal disclosure below CTA (Phase 4)
5. Footer links to Privacy Policy + ToS (Phase 2)
6. Plausible records the pageview + CTA click event (Phase 6)

**Status:** ✓ Complete when user swaps `__STRIPE_CHECKOUT_URL__`.

4. 14-day withdrawal notice visible on success page (Phase 5)

**Status:** ✓ Complete when user swaps `__CALCOM_BOOKING_URL__`.
